## 技巧

### 技巧 1：在開始編碼前使用計劃模式

按下 **Shift+Tab** 切換進入計劃模式。這可以防止 Claude 直接跳入編碼。如果 Claude 過早開始編碼，請按 **Escape** 取消。

> 「把 Claude 想像成一個過於積極的工程師——你真的必須時常約束它。」

你花越多時間規劃，Claude 成功的可能性就越高。在編碼之前一定要先規劃，尤其是對技術背景不深的人來說更是如此。

---

### 技巧 2：請 Claude 與你一起探索解決方案

在計劃模式中，請 Claude 說明程式碼庫並探索多種解決方案，從最簡單的開始。目標是在撰寫任何程式碼之前，先共同制定計劃。

**範例提示詞：**
> 「我想建立一個適合家庭的活動搜尋器。家長可以輸入城市、孩子的年齡、空閒時間、願意開車的距離，以及其他偏好。應用程式將搜尋附近的週末活動，並返回五個推薦結果，每個結果包含粗體標題、相關表情符號，以及兩到四句話的描述。你能從最簡單的方案開始，探索各種解決方案嗎？」

Claude 通常會提出多種方式（例如靜態資料庫、第三方 API、以 Claude Messages API 驅動的 AI 解決方案）。然後你可以選擇最合適的方案。

---

### 技巧 3：遵循規格說明、待辦清單、再編碼的流程

這是 Peter 為每個里程碑推薦的核心工作流程：

1. **規格說明（Spec）** — 請 Claude 建立一份 `spec.md`，內容包含需求、技術堆疊、設計指南，以及最多三個里程碑。審閱並簡化它（Claude 往往傾向於過度建置）。
2. **待辦清單（To-Do）** — 請 Claude 建立一份 `todo.md`，列出當前里程碑的任務。仔細審核清單。
3. **編碼（Code）** — 唯有到這一步，才讓 Claude 開始編碼。

**規格說明的範例提示詞：**
> 「建立一份 spec.md，包含需求、技術堆疊、設計指南，以及最多三個里程碑。盡可能保持簡單。里程碑一：用假資料建立 UI；里程碑二：連接 Claude API 並使用網路搜尋工具。」

你也可以按 **Ctrl+V** 將參考圖片貼入 Claude，給它一個視覺目標。

**待辦清單的範例提示詞：**
> 「根據里程碑一建立一份 todo.md，列出相應任務。」

也可以考慮建立一個 `prompt.md` 檔案，其中包含你想用於 API 呼叫的系統提示詞，讓 Claude 對如何架構 AI 回應有清晰的參考依據。

> 「如果你只記得這個教學中的一件事，那就是遵循規格說明、待辦清單、再編碼的流程。」

### 技巧 4：使用輸出風格在建置的同時學習

輸入 `/output-style`，後接風格名稱，可以改變 Claude 的溝通方式：

- **`/output-style explanatory`** — Claude 會加入說明方塊，解釋為何做出每個決定。非常適合學習技術架構。
- **`/output-style learning`** — Claude 會讓你自己撰寫部分程式碼，以獲得實作練習。

對於想了解正在建置什麼的非工程師來說，這尤其有用。

---

### 技巧 5：使用 CLAUDE.md 快速建立專案理解

在 Claude Code 中輸入 **`/init`**，讓 Claude 掃描程式碼庫並建立一份 `CLAUDE.md` 檔案，其中包含專案背景、模式與關鍵檔案。這個檔案將成為該專案中所有未來對話的 Claude 記憶。

對於尚無任何程式碼的全新專案，你可以建立一個空白的 `CLAUDE.md`，並手動填入偏好設定（見技巧 6）。

---

### 技巧 6：在 CLAUDE.md 中加入個人偏好

將你的背景與偏好加入 `CLAUDE.md`，讓 Claude 根據你的技術水準調整回應。

**CLAUDE.md 的範例內容：**
> 「我是一名程式設計經驗有限的 PM。當你在編碼或執行任務時，請分享技巧，解釋技術架構以及你正在做的更改及原因。」

---

### 技巧 7：使用語音更快速地給 Claude 提供背景資訊

許多頂尖工程師不再打字——他們使用 **Whisper Flow** 或 **Monologue** 等語音工具直接向 Claude 口述。自然說話往往能產生更詳細的指令，有助於 Claude 產生更好的輸出。

你可以按下快捷鍵（例如搭配 Whisper Flow 使用 Ctrl）來口述，然後在傳送前稍微編輯轉錄文字。

---

### 技巧 8：盡快讓你的應用程式運行起來

將你的第一個里程碑設計為能產生一個可見、可運行的應用程式——即使只有假資料也好。這樣你就能在反覆迭代時即時看到更新。

**範例提示詞：**
> 「根據 todo.md 檔案建置里程碑一，然後啟動 localhost。也請使用這張圖片作為參考。」

看到應用程式上線，能讓你更容易向 Claude 提供具體的修復或改進意見。

---

### 技巧 9：按 Escape 停止並重新引導 Claude

在 Claude 編碼的任何時候，你都可以按 **Escape** 來停止它，且不會失去對話背景。以下情況時這點至關重要：

- Claude 走錯方向時
- Claude 做的事超出你的要求時
- Claude 正在安裝不必要的依賴項時
- 你想給它更正確的指示時

> 「不要害怕按 Escape。」

---

### 技巧 10：使用 GitHub 整合進行版本控制

在每個里程碑完成後儲存變更到 GitHub。GitHub 提供版本控制，讓你可以在事情出錯時（使用 AI 一定會出錯的）回退到先前的版本。

**步驟：**
1. 在 GitHub 上建立一個新的儲存庫
2. 在 Claude Code 中，請它將專案連結到該儲存庫並進行初次提交
3. 或者，輸入 `install-github-app` 並按步驟操作

每個里程碑後都提交一次，這樣你永遠有一個安全的回退點。

---

### 技巧 11：設定權限以實現自主工作

Claude 頻繁請求權限，這會拖慢你的速度。你可以建立一個設定檔來自動授予特定權限。

**範例提示詞：**
> 「建立一個設定檔，自動授予讀取、寫入和安全命令的權限。拒絕任何移除或刪除命令。」

這會建立一個 `.claude/settings.json`（或類似的檔案），允許讀取、寫入和 git 命令，但封鎖 `rm` 等破壞性操作。

你也可以使用 `--dangerously-skip-permissions` 標誌，但除非你真的知道自己在做什麼，否則不建議這樣做。

---

### 技巧 12：使用 `/clear` 和 `/compact` 策略性地清除對話背景

隨著對話越來越長，Claude 的表現會下降。使用以下指令來管理對話背景：

- **`/compact`** — 清除對話歷史但保留摘要。這是首選方式，因為 Claude 能保留關鍵背景。
- **`/clear`** — 完全清除對話歷史，不保留任何摘要。

在里程碑之間使用 `/compact`，以保持 Claude 的最佳狀態。

---

### 技巧 13：請 Claude「超深度思考」

在除錯時，在提示詞中加入「think ultra hard（超深度思考）」，讓 Claude 在回應前使用更多 token 並思考更長時間。這會針對複雜問題啟用延伸思考模式。

**範例提示詞：**
> 「我只看到一個空白畫面。請超深度思考為何會發生這種情況。以下是我在主控台看到的錯誤：[貼上錯誤訊息]」

---

### 技巧 14：詢問「你認為這是為什麼發生的？」

這個句子會讓 Claude 深入找出問題根源，而不只是修補症狀。它能迫使 Claude 對出了什麼問題進行更深層的分析。

---

### 技巧 15：提供附有圖片和主控台日誌的具體回饋

Claude 擁有的背景越多，除錯能力就越好。最佳實踐：

- 附上你所看到畫面的截圖（或精確地描述它）
- 複製瀏覽器主控台日誌並貼入 Claude
- 描述預期行為與實際行為的差異
- 具體說明哪裡出了問題（例如：「你顯示的是像 Exploratorium 這樣的通用結果。我想要的是及時且相關的本地活動，例如 Exploratorium After Dark 或 Lindy in the Park。」）

同時要驗證 Claude 是否真的實作了它宣稱已完成的功能——Peter 發現 Claude 在待辦清單上勾選了一個項目（網路搜尋工具），但實際上並未實作。

> 「Claude 就像一個過於積極的初級工程師，聲稱做了某件事，但實際上並沒有。」

---

### 技巧 16：為重複的工作流程建立自訂斜線指令

你可以為你經常重複的工作流程建立自訂 `/斜線` 指令。Peter 有一個專門介紹這個主題的教學。這能節省重複任務所花費的時間。

---

### 技巧 17：建立專業化的子代理

請 Claude 建立擁有各自名稱、描述和提示詞的子代理。每個子代理都有自己的對話背景視窗，因此不會污染你的主要對話。

**使用案例（來自 Anthropic 工程師）：**
- 文件更新
- 安全審計
- 事故回應

主代理決定何時呼叫子代理。

---

### 技巧 18：設定 Hooks 以觸發特定行為

Hooks 讓你能執行在 Claude 完成特定操作後自動觸發的腳本。

**範例：** 當 Claude 完成工作時觸發 Slack 通知。

只需請 Claude 為你建立 hooks——例如：「我想建立一個用於設定 Slack 通知的 hook。」

---

### 技巧 19：安裝 MCP 伺服器以獲得專業化能力

MCP（Model Context Protocol，模型上下文協定）伺服器可為 Claude Code 增加更多能力。安裝方式：

```
claude mcp add <server-name>
```

**實用的 MCP 伺服器：**
- **Serena** — 程式碼搜尋
- **Playwright** — 讓 Claude 能截圖並測試自己的 UI
- **Figma** — 從設計稿到原型的工作流程

**注意：** MCP 伺服器會將額外的背景資訊載入你的提示詞，因此不要同時運行太多伺服器。

---

### 技巧 20：並行運行多個 Claude 工作階段

使用 **git worktrees** 在不同的 Claude 對話中同時建置應用程式的不同部分。例如，同時建置前端和後端。

```
git worktree add <path> <branch>
```

這屬於較進階的操作，適合有經驗的工程師。Peter 在他與 Kieran 的訪談中引用了這項技術的現場示範。
